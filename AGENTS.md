# Agent Guidelines for Taps App

## Build & Development Commands

- **Dev server**: `bun dev` (Expo development server)
- **Type check**: `bun typecheck` (runs tsc)
- **Format check**: `bun lint:format` (Prettier check)
- **Format code**: `bun format:all` (formats entire codebase)
- **Clean**: `bun clean` (removes ios/android dirs)
- Note: No test suite currently configured

## Code Style & Conventions

- **Package manager**: Use `bun` (without "run" for scripts)
- **Formatting**: Tabs (not spaces), no semicolons, double quotes, newlines at EOF
- **Imports**: Include file extensions (.ts/.tsx), use `type` for type-only imports
- **Path aliases**: `@/*` (src), `@assets/*` (assets), `@@/*` (root)
- **TypeScript**: Strict mode enabled, handle array indices safely (noUncheckedIndexedAccess)
- **Styling**: NativeWind/TailwindCSS with `cn()` helper for class merging
- **Components**: Use CVA for variants, follow design-system pattern
- **State**: Jotai atoms (with atomWithStorage for persistence)
- **Assertions**: Use `tiny-invariant` for runtime checks
- **Comments**: Avoid generic/self-explanatory comments, no "generated by" markers
- **Code style**: Match existing patterns in the codebase
